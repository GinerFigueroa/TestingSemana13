¿Qué son?
Conceptos base del Grafo de Flujo de Control

⬤  Nodos

Representan instrucciones, bloques de código o decisiones (if, else). Cada sentencia ejecutable es un nodo.

→  Aristas

Son las flechas que conectan nodos y muestran el flujo de ejecución: paso entre instrucciones o ramas de decisión.

◻  Regiones

Áreas cerradas del grafo (incluida la exterior). Se usan para calcular la complejidad ciclomática.

⟶  Caminos

Secuencias completas de nodos y aristas desde el inicio hasta el fin del programa. Cada ruta posible es un camino.

Tipo 1
Coberturas explicadas con grafos
C1 · Sentencias
Cobertura de Sentencias (Statement Coverage)

Ejecutar cada instrucción del programa al menos una vez. Es el nivel más básico. Equivale a cubrir todos los nodos del grafo.

Cobertura = (nodos ejecutados / nodos totales) × 100
if x > 0: print("Positivo") # ← nodo 2 print("Fin") # ← nodo 3
  • Caso x = 5 → ejecuta nodos 1, 2 y 3 → 100% sentencias
  • Un solo caso puede alcanzar 100% si la ruta TRUE cubre todos los nodos.
  • ⚠️ No garantiza probar la rama FALSE.
Grafo de Flujo
C2 · Decisiones
Cobertura de Decisiones (Branch Coverage)

Cada decisión (if/else, switch) debe tomar ambas ramas: TRUE y FALSE. Siempre requiere ≥ 2 casos.

if x > 0: print("Positivo") # ← rama TRUE else: print("Negativo") # ← rama FALSE
  • Caso 1: x = 5 → entra rama TRUE ✔
  • Caso 2: x = -2 → entra rama FALSE ✔
  • Resultado: 2 casos de prueba → 100% decisiones
Grafo de Flujo
C3 · Condiciones / MCC
Cobertura de Condiciones Múltiples (MCC)

Cada sub-condición booleana debe evaluarse como TRUE y FALSE. Con n sub-condiciones → 2ⁿ combinaciones.

if (a > 0 and b > 0): print("Ambos positivos")
CasoA > 0B > 0Resultado
1FFFALSE
2FTFALSE
3TFFALSE
4TTTRUE
  • 2 sub-condiciones → 2² = 4 casos → MCC 100%
  • MCC es más estricta que decisión (C2) y sentencia (C1)
Árbol de Condiciones
C4 · Caminos
Cobertura de Caminos (Path Coverage)

Recorrer todos los caminos posibles del programa desde inicio hasta fin. Es la más completa pero costosa.

def descuento(monto): if monto > 100: return monto * 0.80 # camino 1 return monto * 0.90 # camino 2
  • Camino 1: monto=150 → IF true → return 0.80
  • Camino 2: monto=50 → IF false → return 0.90
  • Total: 2 casos cubren todos los caminos
  • ⚠️ En programas grandes el número de caminos crece exponencialmente

Tipo 5

Pruebas de Bucles (Loop Testing)

Validan el comportamiento correcto de los ciclos (for, while). Se prueban casos límite y extremos.

Simple

Bucle Simple

Un solo ciclo. Se prueban: 0 iteraciones, 1 iteración, n iteraciones y el valor máximo esperado.

Anidado

Bucle Anidado

Un bucle dentro de otro. Estrategia: probar el interno primero con mínimos del externo, luego combinar.

Concatenado

Bucle Concatenado

Varios bucles en secuencia. Se prueban de forma independiente, como bucles simples individuales.

⬜ Pruebas de Caja Blanca (Estructurales)

Acceso al código. Mide cobertura. La realiza el desarrollador. Verifica la lógica interna.

⬛ Pruebas de Caja Negra (Funcionales)

Sin acceso al código. Verifica requisitos. Puede realizarla un tester externo. Verifica lo que el usuario espera.

🧠 Regla de oro para exámenes C1 (Sentencias) ≤ C2 (Decisiones) ≤ C3 (Condiciones MCC) ≤ C4 (Caminos) en términos de casos requeridos. Si el programa tiene una sola decisión simple, las tres primeras coberturas suelen necesitar el mismo número de casos. La diferencia se nota al tener condiciones compuestas (AND / OR) o múltiples decisiones anidadas.

🧪 Pruebas Estructurales (Caja Blanca)

Resumen completo: cobertura de sentencias (C1), decisión (C2), condición (C3), caminos (C4), bucles y grafos de flujo de control.

📌 Resumen rápido

C1 Cobertura de sentencias → ejecutar todos los nodos (instrucciones) al menos una vez.

C2 Cobertura de decisión → cada decisión (if/else) tome true y false.

C3 Cobertura de condición → cada condición atómica (ej: A>2) sea V y F.

C4 Cobertura de caminos → recorrer todas las rutas posibles del grafo.

✅ Base de análisis: Grafo de flujo de control.

🔹 Nodos → instrucciones o bloques.

🔹 Aristas → flujo de ejecución (flechas).

🔹 Regiones → áreas cerradas (se usan para complejidad ciclomática).

🔹 Caminos → secuencia inicio → fin.

📘 Ejemplos prácticos: ¿cuántos casos de prueba?

🔹 Ejemplo 1 (if simple)

if (a > 2) {
  b = b + 1;
} else {
  b = b - 1;
}
CoberturaCasosExplicación
C1 (sentencias)1Con a=3 se ejecuta todo (incluido el else? No, pero todas las sentencias posibles se ejecutan en una ruta).
C2 (decisión)2a=3 (true), a=1 (false)
C3 (condición)2solo hay una condición atómica (a>2) → V/F
C4 (caminos)2camino true y camino false

🔸 Ejemplo 2 (condición compuesta)

if (x > 3 && y > 0) {
  x = x + y;
} else {
  x = 0;
}
CoberturaCasosExplicación
C1 (sentencias)2necesitas true y false para cubrir ambas ramas
C2 (decisión)2true (5,2) y false (1,2)
C3 (condición)2cada condición atómica V/F: (x>3) y (y>0) con 2 casos basta para cubrir ambas
C4 (caminos)2solo 2 caminos: true y false

⚠️ MCC (múltiple condición) necesitaría 4 casos (todas las combinaciones de V/F).

🧮 ¿Cómo se cuentan los casos de prueba en grafos con bucles?

Regla clave: en presencia de bucles, C1 (sentencias) y C2 (decisión) siguen siendo finitos, pero C4 (caminos) puede ser infinito si el bucle no tiene límite.

En la práctica, para C4 se usan rutas independientes (complejidad ciclomática) y se prueban casos típicos: 0, 1, n iteraciones.

CoberturaNúmero de casosExplicación

🧠 Regla de oro para bucles:

  • C1 y C2 → siempre finitos, se cuentan nodos y decisiones.
  • C4 (caminos) → en bucles se usa complejidad ciclomática para obtener rutas independientes, y luego se añaden casos de borde (0, 1, máximo).
  • Los bucles no estructurados (con goto/break) aumentan la complejidad y no tienen una estrategia estándar.

🧩 Ejercicios difíciles para practicar C1, C2, C3, C4

🔴 Ejercicio 1 (condiciones anidadas)

if (A > 5) {
  if (B < 10) {
    x = 1;
  } else {
    x = 2;
  }
} else {
  x = 3;
}

Pregunta: ¿Cuántos casos para C1, C2, C3 y C4?

📌 Ver solución
  • C1: 2 casos (uno por cada rama principal).
  • C2: 3 casos (A>5 true/false, y dentro B<10 true/false).
  • C3: 3 casos (A>5 V/F, B<10 V/F).
  • C4: 3 caminos (A>5→B<10, A>5→B>=10, A<=5).

🟣 Ejercicio 2 (bucle con condición)

for (i=0; i<10; i++) {
  if (i % 2 == 0) {
    sum += i;
  }
}

Pregunta: C1, C2, C3, C4 (considera el bucle).

📌 Ver solución
  • C1: 1 caso (ejecuta todas las sentencias).
  • C2: 2 casos (if true/false).
  • C3: 2 casos (i%2==0 V/F).
  • C4: infinito (el bucle da muchos caminos). Se prueban 0, 1, 5, 10 iteraciones.

📏 ¿Cómo se cuentan las aristas entre dos nodos?

Regla: cada flecha (arista) se cuenta como una conexión. En el grafo anterior hay 14 aristas.

  • Sentencia (C1): no importa el número de aristas, solo los nodos.
  • Decisión (C2): cada arista que sale de un nodo de decisión cuenta como una rama (true/false).
  • Caminos (C4): las aristas definen las rutas. Número de caminos = combinaciones de aristas.

Fórmula de complejidad ciclomática: V(G) = Aristas - Nodos + 2 → 14 - 10 + 2 = 6 rutas independientes.

✅ En resumen:

  • C1 (sentencias): cubrir nodos → casos = caminos que cubran todos los nodos.
  • C2 (decisión): cubrir aristas true/false → casos = 2 × número de decisiones.
  • C3 (condición): cubrir condiciones atómicas V/F.
  • C4 (caminos): con bucles → infinito, se usan rutas base + casos borde.
  • En grafos con bucles: C1 y C2 son finitos, C4 requiere criterio (0, 1, n, máximo).

📌 Tip: para exámenes, recuerda que “C1 ≤ C2 ≤ C4” en general, y con bucles C4 puede ser mucho mayor o infinito.
